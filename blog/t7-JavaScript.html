Dear Ruby, you've been missed. Oh, hello JavaScript.

2015-06-13

Ruby Class vs. JavaScript Object Constructor

No offense JavaScript, I'm sure you're amazing, it's just that I've been spending the past few weeks with Ruby and I thought we really had something going there for a second. Then you come along and teach me that love is not easy. There is work to do. You're not completely foreign to me, but there are some distinct differences and similarities between you and Ruby. For one thing--objects. Whereas almost everything in Ruby is an object, you are very specific about what is or is not considered an object. Okay, enough talking to you JS, I have thousands of readers (right??) to talk to who are dying to read a bit about what makes you unique from Ruby.

Review of the Ruby Class and Object

Before I dive into the differences between the Ruby Class and JavaScript Constructor Function, I'll quickly review the Ruby Class. A Ruby Class can be considered a collection of properties and methods that instantiate an object. That object then inherits the properties and methods of its class. I read somewhere that you can think of the collection of properties like a Ruby hash, where you can use the instance variables of a class as the key to access the property's value. Something like this:


      class Person

        def initialize(name, age)
          @name = name
          @age = age
        end

        def name
          @name
        end

        def age
          @age
        end

      end

      leon_obj = Person.new("Leon", 27)
      leon_obj.name
      => "Leon"

As you can see, we accessed the Leon object's name by defining it as an instance variable, so it works similar to a method. Ruby uses classes a lot to organize its information about an object, including both attributes and methods. Methods are like functions for an object, they give the object its behavior. Every object in Ruby comes from a class, and there are several built in classes like Fixnum, Object, and String.

The JavaScript Constructor

The JavaScript Constructor is similar to a Ruby Class in that you can create new instances of a JavaScript object from the constructor. Calling a JavaScript constructor function does the following things (and I'm sure more):

  • Creates new objects
  • Sets up a constructor property for the new object
  • Sets the object to respond to the prototype of the constructor

Like the Ruby Class, a JavaScript constructor function also looks like a data structure with properties and functions.


     function Person(name, age) {
        this.name = name;
        this.age = age;
     }

     leon_obj = new Person("Leon", 27);
     leon_obj.name;
     =>'Leon'

This looks super similar to the Ruby Class way to create objects. What's the difference? Well, for one thing, inheritance is handled differently in JavaScript. Instead of class inheritance, JavaScript has a prototype object that is basically an empty object created with each constructor function.

Also, notice this this keyword? This is similar to self in Ruby, and it's saying whichever object is in focus, apply this variable assignment.

Back to the prototype, from what I can tell so far, it looks like you can use the prototype property of a constructor function to modify the constructor.


function Person(name, age) {
  this.name = name;
  this.age = age;
 }

Person.prototype = {
  sayHi: function sayHi() {
      return "Hello!";
   }
}

leon_obj.new Person("Leon", 27);
leon_obj.sayHi();
=> "Hello!"
