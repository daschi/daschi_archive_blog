Ruby Arrays & Hashes

This past week I've delved into learning about Arrays and Hashes in Ruby, and learned that there's a lot of opinions about using one or the other, or both. I'll present here what I've learned so far, but I know I'll need to revisit this blog when I see both their uses on regular basis.
As a quick intro, Arrays and Hashes are two types of data structures that allow you to sort and search through a list of elements (ANY list, a list of words, numbers, mix of the two, or even a list of a list of a list).

Arrays

There are several ways to create an array in Ruby, and it's good to note that each element in an array can be a different data type:

  1. array_name = Array.new

    2. Using the array new method

  2. array_name = ["item 1", 24, true]

    3. Using the array literal notation with square brackets

  3. %w{This is an array}

    4. Using the %w operator, the above will create the array ["This", "is", "an", "Array"]

A simple array looks like this:

ice_cream = ["chocolate", "vanilla", "strawberry", "cookies & cream"]

This array has 4 elements, if we wanted to know which ice cream was third in the list, we could check at the index 2. Arrays in Ruby start counting elements from 0.

ice_cream[2]
=>"strawberry"

There's a ton of different methods you can use to manipulate and iterate over an array! You can insert (.push), retrieve (array[0]) or remove (.delete_at() any element in an array. You can also retrieve 1 or more array elements at a time. You can also combine two or more different arrays into one, which can give you an array within an array, or a "flattened" array, where all elements are taken our of each of their own individual arrays and added together to make one array. The element .join will string together string elements, and alternatively, the .split method will take a string and split it into an array based on a specified character divider. Iterating over an array means you can define a method to look at each element and do something with it.

Hashes

Just like the Array, the hash is what's considered a "collection" in Ruby, meaning that it's a collection of elements that can be accessed and manipulated in different ways. A hash is set up a bit differently than an array, instead of a list of elements, it's a list of key/value pairs. The key/value pair's key is used to quickly identify access the value needed. Check out this great explanation of the difference between accessing a value in an Array vs. a hash!

To create a new hash, you can use similar methods to creating an array:

  1. hash_name = Hash.new(default value)

    2. Using the hash new method with a default value specified in the argument will give any key that is not paired with a value the default value.

  2. Hash[[key,value], [key2,value2]]

    3. Hash can use the inner arrays to create the key/value pairs, or, it can also take a list of elements and assign the left element as key and right element as value. If there's an odd number of elements, it will produce an error.

  3. hash = {}

    4. This creates an empty hash that you can then push the key/value pairs into. To push the pairs, you would use the following syntax: hash["key"] = "value"

A simple hash:

food_stuffs = ["ice cream" => "chocolate", "sandwich" => "BLTA", "soup" => "potato leek"]

This hash has 3 key/value pairs, if we wanted to know which value corresponds with which key, would could check by calling that key. Unlike an Array, a Hash doesn't necessarily sort the key/value pairs in any order because they can be accessed by the key instead of the index position

food_stuffs["ice cream"]
=>"chocolate"

Just like arrays, you can retrieve information from a hash, select key/value pairs within a hash, and transform a hash by inverting it, clearing it, rejecting elements, or replacing the contents of the key/value pairs. You can also combine hashes.

One thing that is clearly different between an array and a hash is the reason you would iterate over it. For an array, you are iterating over each element to find the value you need. In a hash, the key tells you the value almost instantly, so you don't necessarily need to iterate over a hash to find a value. Iterating over a hash will give you the key/value pairs.
