Ruby Enumerable Method group_by

Enumerable#group_by

2015-05-24

Grouping Collections by Anything--seriously?

The group_by method in Ruby takes a collection (like a Hash, Array or Range) and divides it into different groups depending on the criteria given in the code block that follows the method.

example_array.group_by {|item_in_list| criteria here}

The criteria can be anything you want to use to group the items in the list! The group_by method will return a Hash with keys displaying the grouping criteria, and the values matching that criteria from the list.


Let's look at some examples!

  1. Array: group_by word length

    2. names = ["Daniela", "Leon", "Drake", "Alexis", "Tom", "Paul"]
names.group_by {|name| name.length}

=> {7=>["Daniela"], 4=>["Leon", "Paul"], 5=>["Drake"], <br>6=>["Alexis"], 3=>["Tom"]}

names.group_by {|name| name.length > 4}
=> {true=>["Daniela", "Drake", "Alexis"], false=>["Leon", "Tom", "Paul"]}

    The first example here shows that the array of names is returned as a hash, with the group by criteria as the key and the corresponding values. For example, in the second pair, the name length "4" is the key and the names with 4 letters each are the values.

    In the second example, we passed in a code block that evaluates to either True or False. Here you see the names separated by names whose lengths are greater than 4 (True) and those less than 4 (False).

  2. Array: group_by include?

    3. names = ["Daniela", "Leon", "Drake", "Alexis", "Tom", "Paul"]

names.group_by{|name| name.include?("a")}
=> {true=>["Daniela", "Drake", "Paul"], false=>["Leon", "Alexis", "Tom"]}

    This shows another example where the code block evaluates to True or False. Here we're asking each name in the list whether or not it includes the letter "a". If yes, then it's grouped with the True pile, otherwise it's sanctioned to the lonely False planet devoid of the letter "a" (Sorry Leon!).

  3. Range: group_by value, divisibility, and so on

    4. With ranges, you can divide a range of numbers by any numerical criteria you want. Here I chose dividiable by 2, but it could be anything. You can see that the keys mark the two groups, where 1 lists the numbers with a remainder of 1 when divided by 2, and the 0 group holds the numbers with remainder of 0 when divided by 2. If we had set this to divisible by 3, we would have 3 groups (remainder 0, 1, and 3 groups).

    (1..6).group_by {|x| x % 2}
=> {1=>[1, 3, 5], 0=>[2, 4, 6]}

  4. Hash: group_by value or key

    5. names = {"Daniela" => "cool", "Alexis" => "cool", "Leon" => "OK", <br>"Drake" => "OK", "Paul" => "OK"}
names.group_by {|k, v| v == "cool"}
=> {true=>[["Daniela", "cool"], ["Alexis", "cool"]], false=>[["Leon", "OK"], <br>["Drake", "OK"], ["Paul", "OK"]]}

    Here we are grouping based on the value. If the value equals cool in the key/value pair, then it's grouped in the True pile. Seems like we can easily tell who the cool kids are in this group...

    Moving on...you can also group by first letter, second letter, [x] letter, basically, in either an Array, Hash or Range, you can group by any specific property you can call out from the list.

    names.group_by {|k, v| k[0]}
=> {"D"=>[["Daniela", "cool"], ["Drake", "OK"]], "A"=>[["Alexis", "cool"]], <br>"L"=>[["Leon", "OK"]], "P"=>[["Paul", "OK"]]}

    This here is grouping by 1st letter of the key, as shown by k[0], which is asking for the zero index of the key. The zero index of "Daniela" is "D".

Conclusion

The group by method can be seen as a delegator of items based on the criteria you want to divide along. It can tell you who's cool or just OK, it can group by divisibile remainders, or really any criteria you give it that is an actual property or some or all of the items in the list. I can see this being extremely useful when you want to sort through a list and group them into different categories for different purposes.
