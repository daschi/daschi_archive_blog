OK Recap!--Let's talk version control. What are the benefits?

  • Collaborate on projects with other people and work simulteanously
  • Track and merge all major changes to a file from multiple people
  • Keep a master version unpolluted by funky code
  • Test different versions before declaring it final

The way I see it, a world without version control would be a very scary, unproductive and confusing place, with hundreds of local versions of the same files and no safe or efficient way to ever merge changes!

This would be like sending an essay to 3-4 people to critique, and then receiving dozens of mysterious, untracked changes. You'd have to compare 4 different versions and find all the changes yourself! How awful! Fortunately, version control has made it so developers will never need to experience such woes.

So how do you implement version control you ask? You use a version control system like Git!

Git is a software that lives on your local machine and you can use it within your terminal. It helps you keep track of your changes by creating branched versions off of your master copy. Your master is just as it sounds--the very final version of whatever you're working on. You can work on each branch separately and then merge them all together into the master when they're absolutely ready. There are a few key commands you need to use git:

  1. git checkout -b [name]

    2. The [name] placeholder is where you would put your branch name. This command creates a new branch from your master version.

  2. git status

    3. This command shows whether any of the files or directories in your branch have any changes, and in what stage they're in for the version control process. They can either be "untracked", "added", or "staged to commit". When you commit the changes, they disappear from your status log.

  3. git add

    4. This adds a file so that it's 'staged to commit' and the changes can be tracked and committed as final.

  4. git commit

    5. This step saves your changes and updates your local repository with the new version.

So why use GitHub?

GitHub is an online Git repository hosting service where you can push and store your local git versions. This is useful so you can work together with others on the same project, and everything is stored in the cloud, making it more accessible. GitHub also provides a social networking and community-driven environment for reviewing and collaborating on projects.
